what is koll♠b

koll♠b is a transparent, adversarial-collaborative dialogue engine that pits two AI coding agents — Claude (Anthropic) and Codex (OpenAI) — against each other on a single goal you define. Every turn, every critique, every revision, and every disagreement is rendered live in your browser as it happens.

The product is the visibility into the inter-model dynamic — not the orchestration itself. koll♠b is a demo and observability tool, not a production pipeline. The goal is legibility: you should be able to follow exactly what each model understood, what it challenged, and why it changed its mind (or didn't).

koll♠b runs entirely on your machine. No cloud, no accounts, no telemetry. Session logs are stored as plain JSONL files on disk. Nothing leaves your computer except the API calls to Anthropic and OpenAI.

how it works — enter ACE, the Adversarial Collab Engine

At the heart of koll♠b is ACE — the Adversarial Collab Engine. ACE owns the session state machine, drives every turn, parses verdict trailers, and enforces the convergence rules. It doesn't understand what Claude and Codex are saying — it reads the <verdict> tag at the end of each turn and acts on it mechanically. A single Codex AGREE is the sole convergence signal — ACE closes the session immediately, regardless of round count. When the round limit is hit without agreement, ACE stops the loop. The intelligence is in the agents; ACE runs the show.

When you submit a goal, ACE fans it out to both agents simultaneously — each model reads your goal first-hand and forms its own understanding before seeing any peer output. ACE then alternates turns, feeding each agent only the peer's latest message (not a full transcript replay), so each model reacts to the other in real time, the way a genuine debate works.

Claude is the Producer. It writes the initial proposal — code, design, argument — and defends or revises it in response to critique. Claude's first turn is always labeled PROPOSAL.

Codex is the Critic. It adversarially reviews Claude's output, finds real flaws, and issues a verdict each turn. When Codex is satisfied, the session converges. The critic's verdict is the only one that matters — a single Codex AGREE ends the session, regardless of round count.

Each turn ends with a structured verdict trailer that ACE parses:

AGREE critique accepted / issue resolved — debate over
DISAGREE position held, critique rejected — continue
REVISED work updated in response to critique — continue

Sessions end on convergence (Codex AGREEs), round limit (max rounds without agreement), token limit (session budget exhausted), or halt (you stopped it).

how to use

The main interface

The koll♠b interface: history pane (left), session tab bar (top), dialogue area (centre), status strip and input bar (bottom). Top-right: ?, theme toggle, Configure, Quit.

The left history pane lists all past sessions — click ≡ to collapse or expand it. The main area shows the active session's turn cards streaming in real time. The top-right cluster holds global controls: ? opens this page, the ☀/🌙 toggle switches between light and dark theme, ⚙ Configure opens global settings, and ⏻ Quit shuts the server down cleanly.

History pane (≡)

The history pane showing 34 sessions filtered by outcome. The highlighted row is the currently open tab.

Every completed session appears here with a goal preview, relative timestamp, and an end-reason label. The three coloured filter squares at the top let you filter by outcome — ■ green (converged), ■ amber (round limit), ■ red (halted or expired). Click any row to open that session as a read-only tab. Hover a row to reveal the × delete button — deletion permanently removes the session log from disk.

+ New Session

The New Session modal. Enter your goal, optionally override models and limits for this run.

Click + New Session in the top-right of the tab bar to open this modal. Describe your goal in the text area — a coding problem, a design question, an algorithm to review — then optionally override the defaults just for this run:

  • Claude model / Codex model — pick the tier for each agent (haiku/sonnet/opus, mini/gpt-5.4)
  • Round limit — cap the number of back-and-forth rounds (default from Configure)
  • Max tokens / turn — advisory limit injected into each agent's prompt
  • Max tokens / session — hard halt when cumulative token budget is hit

Only one session can run at a time. The Start button is disabled if a session is already active.

⚙ Configure

The Configure modal showing all global settings in a two-column layout.

Global settings that apply to every session unless overridden at session start:

  • Claude / Codex binary path — path to the CLI binaries (default: claude, codex)
  • Claude / Codex model — default model for new sessions
  • Claude / Codex working dir — the directory each agent operates in
  • Round limit — default max rounds per session
  • Halt timeout — seconds before a halted session auto-expires (0 = never)
  • Port — server port (restart required to take effect)
  • Sessions dir — where JSONL session logs are stored
  • Filesystem MCP — gives Claude read access to specified paths; leave blank to use its working dir
  • API key — bearer token for the REST API; empty means no auth (safe for local use only)
  • Webhooks — enable outbound event delivery: HTTP targets, Slack incoming webhook URLs, per-event toggles, and delivery timeout

GitHub MCP is coming in a future release.

Turn cards

[ screenshot: turn cards — coming ]

Claude turn cards have an orange border; Codex cards have a blue border.

Each agent response is a turn card. Orange border = Claude (producer), blue border = Codex (critic). The card header shows the actor name, role, turn ID (C-1, X-1, C-2…), a thread session badge, and the verdict pill. Claude's first card always shows PROPOSAL. Subsequent cards show AGREE, DISAGREE, or REVISED once the turn completes.

The › chevron on the left of each card header collapses the entire card body. If Claude reasoned before responding, a collapsible Reasoning block appears above the result — it's expanded by default so you can follow Claude's thinking as it streams in. Use the − collapse all toggle below the goal card to collapse every card at once, useful for scanning verdicts across a long session.

Stop & Resume

[ screenshot: Stop / Resume — coming ]

The Stop button appears while a session is running. Resume appears when halted.

Stop halts the session at any point — even mid-stream while an agent is responding. The partial output stays visible in the UI as an observable artifact, but it is never fed back to either agent. The session stays open and can be resumed.

On Resume: if you have something to say to an agent before continuing, select a target (Claude, Codex, or both) and type your instruction — this is called a directive. The targeted agent receives it as part of their next prompt. If you resume without a directive, the interrupted agent re-runs fresh from clean history as if the interruption never happened.

Directives

[ screenshot: directive input — coming ]

The input bar with target selector — only enabled when a session is halted.

While a session is halted, the input bar unlocks. Select a target — → Claude, → Codex, or → Claude, Codex — then type your instruction and click Send (or press Enter). The directive is injected into that agent's next turn prompt only; the other agent doesn't see it unless you target both. Directives appear in the session as DIRECTIVE → CLAUDE cards, visible in both live and read-only history views.

Tab bar & × Close all

Each session opens in a tab in the tab bar at the top of the main area. Completed and halted tabs persist across browser refreshes — they're restored automatically on next load from the session log. The × button to the left of the session count closes all inactive tabs at once; the active running session (if any) is never touched. Closing an active tab prompts confirmation and stops the session.

− Collapse all / + Expand all

A small toggle that appears just below the goal card at the start of every session. Click it to collapse every turn card body simultaneously — useful when a session has many rounds and you want to scan just the verdict pills without scrolling through full responses. Click again to expand everything back.

stack & design philosophy

Python 3.11+, FastAPI, WebSockets on the backend. Vanilla HTML + Tailwind CDN + a single app.js on the frontend. No build step, no bundler, no React. JSONL logs on disk, TOML config — no database, no multi-user auth, no multi-tenancy. The REST API supports optional bearer token auth for headless callers.

koll♠b is a demo and observability tool. Every design decision optimises for legibility and demo-ability over scale or robustness. If a suggestion starts sounding enterprise-grade, it's wrong for this project.
